    **Create WishlistRepository domain interface:**
    Create `domain/src/main/java/com/wenubey/domain/repository/WishlistRepository.kt`:
    ```kotlin
    interface WishlistRepository {
        fun observeWishlistItems(userId: String): Flow<List<WishlistItem>>
        fun isWishlisted(userId: String, productId: String): Flow<Boolean>
        suspend fun toggleWishlist(userId: String?, product: Product)
        suspend fun removeFromWishlist(userId: String, productId: String)
        suspend fun syncAnonymousOnLogin(userId: String)
    }
    ```
    - `userId` is nullable in `toggleWishlist` to support anonymous users — impl uses "" when null
    - Import `WishlistItem` from domain model, `Product` from domain model, `Flow` from kotlinx.coroutines

    **Create WishlistRepositoryImpl:**
    Create `data/src/main/java/com/wenubey/data/repository/WishlistRepositoryImpl.kt`:
    Constructor params: `WishlistItemDao`, `ProductDao` (for stock refresh), `FirebaseFirestore`, `DispatcherProvider`

    **observeWishlistItems(userId):**
    Return `wishlistItemDao.observeWishlistItems(userId).map { entities -> entities.map { it.toDomain() } }`. Flowable, reactive.

    **isWishlisted(userId, productId):**
    Return `wishlistItemDao.isWishlisted(userId, productId)`. This is a Flow<Boolean>.

    **toggleWishlist(userId, product):**
    1. Determine effective userId: `val effectiveUserId = userId ?: ""`
    2. Check if wishlisted: `val existing = wishlistItemDao.getWishlistItem(effectiveUserId, product.id)`
    3. If exists (removing from wishlist):
       - `wishlistItemDao.deleteItem(effectiveUserId, product.id)`
       - If effectiveUserId is not "" (logged-in user): delete Firestore document `users/{userId}/wishlist/{productId}` — wrap in try/catch, log error but don't fail (Room is source of truth)
    4. If not exists (adding to wishlist):
       - Create WishlistItemEntity: userId=effectiveUserId, productId=product.id, productTitle=product.title, productImageUrl=product.images.firstOrNull()?.url ?: "", productPrice=product.basePrice, availableStock=product.totalStockQuantity, isProductDeleted=false, addedAt=Instant.now().toString()
       - `wishlistItemDao.upsert(entity)`
       - If effectiveUserId is not "" (logged-in user): write to Firestore `users/{userId}/wishlist/{productId}` with {productId, productTitle, productPrice, addedAt} — wrap in try/catch
    5. No PendingOperationEntity for wishlist — wishlist sync is simpler (direct Firestore write if online, Room is persistent regardless). This avoids overcomplicating the queue for a non-critical feature. If offline, the Room write persists and can sync on next login or manual refresh.

    **removeFromWishlist(userId, productId):**
    Same as the "exists" branch of toggleWishlist — delete from DAO and Firestore.

    **syncAnonymousOnLogin(userId):**
    Per research Pattern: Anonymous wishlist sync on login.
    1. `val anonymousItems = wishlistItemDao.getItemsForUser("")`
    2. If empty, return early
    3. For each item:
       a. `wishlistItemDao.upsert(item.copy(userId = userId))` — migrate to real userId in Room
       b. Write to Firestore `users/{userId}/wishlist/{item.productId}` document with item data — use set() for upsert semantics (merges if doc exists from another device)
    4. After all items migrated: `wishlistItemDao.deleteAllForUser("")` — clean up anonymous rows
    5. Also fetch existing Firestore wishlist items and upsert to Room (merge strategy):
       a. `firestore.collection("users").document(userId).collection("wishlist").get().await()`
       b. For each Firestore doc, create WishlistItemEntity and `wishlistItemDao.upsert(entity)`
    6. Wrap in `withContext(dispatcherProvider.io())` and try/catch for Firestore errors

    **Where to call syncAnonymousOnLogin:**
    This should be called from `AuthRepositoryImpl` after successful login. Add a note in the action that the caller wiring is handled in this task:
    - In `AuthRepositoryImpl`, after the auth state listener detects a new login (user becomes non-null), call `wishlistRepository.syncAnonymousOnLogin(user.uid)`. This may require adding WishlistRepository as a dependency to AuthRepositoryImpl, or using a coroutine callback pattern.
    - Alternatively, call it from `AuthViewModel` in the app module after login succeeds. This is simpler and avoids circular dependencies. **Prefer the AuthViewModel approach** — after sign-in/sign-up success, before navigating to main screen, call `wishlistRepository.syncAnonymousOnLogin(userId)`.
    - Since AuthViewModel is in the app module and WishlistRepository is in domain, this wiring is clean. Add `WishlistRepository` to `AuthViewModel` constructor and call sync after login detection.

    Note: Modify `AuthViewModel` to inject `WishlistRepository` and call `syncAnonymousOnLogin` when user state transitions from null to non-null. Wrap in `viewModelScope.launch` with try/catch — sync failure should not block login.
