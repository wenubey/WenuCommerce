    **CheckoutState.kt:**
    ```kotlin
    data class CheckoutState(
        val currentStep: Int = 0,  // 0=Address, 1=Review, 2=Payment
        val cartItems: List<CartItem> = emptyList(),
        val savedAddresses: List<ShippingAddress> = emptyList(),
        val selectedAddress: ShippingAddress? = null,
        val subtotal: Double = 0.0,
        val shippingTotal: Double = 0.0,
        val total: Double = 0.0,
        val clientSecret: String = "",
        val orderId: String = "",
        val amountCents: Int = 0,
        val isCreatingPaymentIntent: Boolean = false,
        val isProcessingPayment: Boolean = false,
        val paymentError: String? = null,
        val stockError: String? = null,  // server-side stock check failure message
        val isOnline: Boolean = true,
        val isLoading: Boolean = true,
    ) {
        val canProceedToReview: Boolean
            get() = selectedAddress != null
        val canProceedToPayment: Boolean
            get() = clientSecret.isNotEmpty() && !isCreatingPaymentIntent
    }
    ```

    **CheckoutAction.kt:**
    ```kotlin
    sealed interface CheckoutAction {
        data class SelectAddress(val address: ShippingAddress) : CheckoutAction
        data object NavigateToAddAddress : CheckoutAction
        data class GoToStep(val step: Int) : CheckoutAction
        data object NextStep : CheckoutAction
        data object PreviousStep : CheckoutAction
        data object CreatePaymentIntent : CheckoutAction
        data object RetryPayment : CheckoutAction
        data object DismissPaymentError : CheckoutAction
        data object DismissStockError : CheckoutAction
    }
    ```

    **CheckoutViewModel.kt:**
    - Extends ViewModel. Constructor injects: `PaymentRepository`, `CartRepository`, `AddressRepository`, `AuthRepository`, `ConnectivityObserver`.
    - `_state = MutableStateFlow(CheckoutState())`; public `state = _state.asStateFlow()`
    - `init {}`:
      1. Collect `connectivityObserver.isOnline` -> update `_state { copy(isOnline = it) }`
      2. Get current userId from `authRepository.currentUser.value?.uuid`
      3. Collect `cartRepository.observeCartItems(userId)` -> update cartItems, compute subtotal (same logic as CartState.subtotal: filter available, sumOf price*qty)
      4. Compute shippingTotal: sum of per-product shipping costs. Since CartItem does not carry shipping cost, pass `shippingCost = 0.0` for now and let the Cloud Function be the source of truth. Client-side: compute subtotal only; total shown after PaymentIntent response.
         - ALTERNATIVE: If Product model is accessible, look up shipping from product. For simplicity, show "Subtotal + Shipping (calculated at checkout)" on Review step, then show the confirmed total from `amountCents` after PaymentIntent creation.
      5. Collect `addressRepository.observeSavedAddresses(userId)` -> update savedAddresses. Auto-select first address if available.
      6. Set `isLoading = false`

    - `onAction(action: CheckoutAction)`:
      - `SelectAddress`: update selectedAddress
      - `GoToStep(step)`: Only allow going backward (step < currentStep) freely. Going forward: step 0->1 requires selectedAddress != null; step 1->2 triggers createPaymentIntent.
      - `NextStep`: Increment currentStep. On transition from step 1 to step 2, auto-call `createPaymentIntent()`.
      - `PreviousStep`: Decrement currentStep (min 0).
      - `CreatePaymentIntent`: Launch in viewModelScope on IO dispatcher. Set `isCreatingPaymentIntent = true`. Call `paymentRepository.createPaymentIntent(userId, cartItems, selectedAddress!!)`. On success: update `clientSecret`, `amountCents`, `orderId`, set `isCreatingPaymentIntent = false`. On failure: set `stockError` or `paymentError` with error message, move back to step 1 (Review) if it's a stock error.
      - `RetryPayment`: Clear paymentError, re-call createPaymentIntent.
      - `DismissPaymentError`/`DismissStockError`: Clear respective error fields.

    - `onPaymentResult(result: PaymentSheetResult)`: Public method called from composable.
      - `Completed`: Set `isProcessingPayment = true`. Launch coroutine: call `paymentRepository.createOrderInRoom(order)` to persist the order locally (build Order from state). Call `cartRepository.clearCart(userId)`. Then signal navigation to OrderConfirmationScreen (via a `NavigationEvent` Channel or `orderId` flag in state).
      - `Canceled`: No-op, user stays on step 3.
      - `Failed`: Set `paymentError = result.error.message ?: "Payment failed"`, `isProcessingPayment = false`.

    - **Navigation signal for order confirmation:** Use a `SharedFlow` or `Channel<String>` pattern: `val navigationEvent: SharedFlow<String>` that emits `orderId` when checkout completes. The screen collects this and navigates.

    **ViewmodelModule.kt:** Add `viewModelOf(::CheckoutViewModel)`.

    **IMPORTANT (from research Pitfall 4):** The `rememberPaymentSheet` callback must NOT capture ViewModel state. It should only call `viewModel.onPaymentResult(result)`.

    **IMPORTANT (from research Pitfall 6):** CheckoutViewModel is obtained via `koinViewModel()` in CheckoutScreen. Since CheckoutScreen is a single navigation destination, the ViewModel is scoped to that destination's lifecycle. When the user navigates away (back to cart or to confirmation), the ViewModel is cleared. This is correct behavior.
