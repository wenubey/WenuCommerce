    **OrderConfirmationScreen.kt:**
    - Parameters: `orderId: String, onContinueShopping: () -> Unit, onViewOrder: (String) -> Unit`
    - Use `koinInject<PaymentRepository>()` to load the order: `LaunchedEffect(orderId) { order = paymentRepository.getOrderById(orderId) }`
    - Layout (centered Column):
      1. **Animated checkmark**: Use `LottieAnimation` with `rememberLottieComposition(LottieCompositionSpec.RawRes(R.raw.success_checkmark))` and `animateLottieCompositionAsState(composition, iterations = 1)`. Size: 120.dp. If Lottie unavailable, use a green check Icon with scale-in animation.
      2. **"Order Placed!" heading**: `Text` with `headlineMedium` style, bold.
      3. **Order ID**: `Text("Order #${orderId.take(8)}...")` with `bodyMedium` style, secondary color. Show truncated ID with copy-friendly full display.
      4. **Item count + Total**: `Text("${order?.items?.size ?: 0} items | Total: ${"$%.2f".format(order?.totalAmount ?: 0.0)}")`.
      5. **Spacer**.
      6. **"Continue Shopping" button** (filled): calls `onContinueShopping()`.
      7. **"View Order" button** (outlined): calls `onViewOrder(orderId)`.
    - Loading state: While order is null and we haven't errored, show a brief loading indicator.

    **MinimalOrderScreen.kt:**
    - Parameters: `orderId: String, onNavigateBack: () -> Unit`
    - Use `koinInject<PaymentRepository>()` to observe the order: `val order by paymentRepository.observeOrderById(orderId).collectAsStateWithLifecycle(null)`
    - Scaffold with TopAppBar ("Order Details", back arrow).
    - Content (when order is loaded):
      1. **Status badge**: Chip or Text showing `order.status.displayName` with color coding (PENDING=amber, CONFIRMED=blue, SHIPPED=purple, DELIVERED=green, CANCELLED=red).
      2. **Order ID**: Full order ID text.
      3. **Date**: `order.createdAt` formatted.
      4. **Items list**: `LazyColumn` with each item: product name, quantity, line total. Compact layout (no images, consistent with review step).
      5. **Totals**: Subtotal, Shipping, Total. Same format as review step.
      6. **Shipping address**: Card with full address details.
    - This is a minimal placeholder — Phase 6 (Order Tracking & Management) will enhance with status timeline, tracking number, seller actions, etc.

    **TabNavRoutes.kt** — Add routes (OrderConfirmation and OrderDetail nav objects were added in Plan 03):
    ```kotlin
    composable<OrderConfirmation> {
        val args = it.toRoute<OrderConfirmation>()
        OrderConfirmationScreen(
            orderId = args.orderId,
            onContinueShopping = {
                navController.navigate(CustomerTab(tabIndex = 0)) {
                    popUpTo<CustomerTab> { inclusive = true }
                }
            },
            onViewOrder = { orderId ->
                navController.navigate(OrderDetail(orderId))
            },
        )
    }
    composable<OrderDetail> {
        val args = it.toRoute<OrderDetail>()
        MinimalOrderScreen(
            orderId = args.orderId,
            onNavigateBack = { navController.popBackStack() },
        )
    }
    ```

    - Checkout -> PaymentSheet -> Completed -> `OrderConfirmation(orderId)` route (pops back to CustomerTab)
    - OrderConfirmation -> "Continue Shopping" -> `CustomerTab(0)` (home)
    - OrderConfirmation -> "View Order" -> `OrderDetail(orderId)`
    - OrderDetail -> back -> OrderConfirmation (or wherever user was)

    **Verify the entire navigation chain compiles and the back stack is clean:**
    - After checkout completes and user goes to confirmation, pressing back should NOT return to checkout (it was popped).
    - From confirmation, "Continue Shopping" goes to home tab.
    - From confirmation, "View Order" goes to order detail; back returns to confirmation.
