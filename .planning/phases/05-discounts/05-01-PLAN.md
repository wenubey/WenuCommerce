  <name>Task 2: Cloud Functions -- validateCoupon, decrementCouponUsage, and createPaymentIntent coupon extension</name>
  <files>functions/src/index.ts</files>
  <action>
    1. Add shared helper functions at module level (above createPaymentIntent):

       a. `computeDiscount(couponData, cartItems, subtotalCents, shippingCents)`:
          - PERCENTAGE: `Math.min(Math.round(subtotalCents * couponData.value / 100), couponData.maxDiscountCap ? Math.round(couponData.maxDiscountCap * 100) : Infinity)` -- only apply to eligible items if targetProductIds is non-empty
          - FIXED_AMOUNT: `Math.round(couponData.value * 100)` -- capped at subtotalCents
          - FREE_SHIPPING: returns `shippingCents`
          - For product-scoped coupons (targetProductIds.length > 0): compute discount only on the subtotal of eligible items (filter cartItems by productId membership in targetProductIds)

       b. `validateCouponData(data, cartItems, subtotalCents)`:
          - Shared validation logic used by both validateCoupon and createPaymentIntent
          - Checks: isActive, expiresAt, usageLimit vs usageCount, eligible items (targetProductIds), minimumOrderCents
          - Throws HttpsError with specific codes: "not-found" / "Code not found", "failed-precondition" / "This code has expired", "resource-exhausted" / "Usage limit reached", "failed-precondition" / "No eligible items in your cart", "failed-precondition" / "Minimum order of $X.XX required"

    2. Create `validateCoupon` onCall Cloud Function:
       - `export const validateCoupon = onCall({ secrets: [stripeSecretKey] }, async (request) => { ... })`
       - Auth check (same pattern as createPaymentIntent)
       - Extract `couponCode`, `cartItems`, `subtotalCents` from request.data
       - Normalize code: `couponCode.trim().toUpperCase()`
       - Fetch document: `db.collection("discountCodes").doc(normalized).get()` (O(1) lookup by doc ID)
       - If !doc.exists: throw HttpsError("not-found", "Code not found")
       - Call `validateCouponData(data, cartItems, subtotalCents)`
       - Call `computeDiscount(data, cartItems, subtotalCents, 0)` (shippingCents=0 for preview)
       - Return `{ code: normalized, type: data.type, discountCents, description }` where description is human-readable (e.g., "20% off", "$10.00 off", "Free shipping")

    3. Create `decrementCouponUsage` onCall Cloud Function:
       - Auth check
       - Extract `couponCode` from request.data
       - Normalize to uppercase
       - `db.collection("discountCodes").doc(normalized).update({ usageCount: admin.firestore.FieldValue.increment(1), updatedAt: admin.firestore.FieldValue.serverTimestamp() })`
       - CRITICAL: Use `FieldValue.increment(1)` (NOT read-modify-write) -- atomic, race-condition-free
       - Return `{ success: true }`

    4. Extend `createPaymentIntent`:
       - After computing `subtotalCents` and `shippingCents`, check for optional `request.data.couponCode`
       - If present: normalize, fetch coupon doc, call `validateCouponData`, call `computeDiscount` with actual shippingCents
       - Apply: `const finalTotalCents = Math.max(50, subtotalCents + shippingCents - discountCents)` (Stripe minimum is 50 cents)
       - Pass `finalTotalCents` (not `totalCents`) to `stripe.paymentIntents.create`
       - Add to Firestore order document: `discountAmount: discountCents / 100`, `discountCode: appliedCouponCode || ""`
       - Return `discountAmountCents: discountCents` alongside existing clientSecret/amountCents/orderId
       - If couponCode is absent or empty, behavior is identical to current (no discount logic runs)
