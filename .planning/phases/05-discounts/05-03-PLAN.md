    2. Extend `CheckoutAction` sealed interface:
       ```kotlin

       data object DismissCouponError : CheckoutAction
       ```

    3. Add `DiscountRepository` to `CheckoutViewModel` constructor (inject via Koin -- already wired in plan 01).

    4. Handle new actions in `onAction()`:
       - `UpdateCouponInput`: update `couponInput` in state, clear `couponError`
       - `ApplyCoupon`: call `applyCoupon()` (new private method)
       - `RemoveCoupon`: call `removeCoupon()` (new private method)
       - `DismissCouponError`: clear `couponError` in state

    5. Implement `applyCoupon()`:
       - Set `isValidatingCoupon = true`, clear `couponError`
       - Compute subtotalCents from cartItems: `(subtotal * 100).toInt()`
       - Call `discountRepository.validateCoupon(couponInput, cartItems, subtotalCents)`
       - On success: update state with `appliedCouponCode = result.code`, `appliedCouponType = result.type`, `discountAmountCents = result.discountAmountCents`, `isValidatingCoupon = false`, `couponInput = ""`
       - On failure: update state with `couponError = error.message` (already mapped to user-facing string by DiscountRepositoryImpl), `isValidatingCoupon = false`

    6. Implement `removeCoupon()`:
       - CRITICAL (Pitfall 4 from research): When coupon is removed, the existing clientSecret was created with the discounted amount. Must invalidate it.
       - Update state: `appliedCouponCode = null`, `appliedCouponType = null`, `discountAmountCents = 0`, `clientSecret = ""`, `orderId = ""`, `amountCents = 0`
       - Keep `isExpanded` as local composable state (Pitfall 2) -- ViewModel does not control collapse

    7. Update `createPaymentIntent()`:
       - Pass `state.appliedCouponCode` to `paymentRepository.createPaymentIntent(userId, cartItems, address, couponCode = state.appliedCouponCode)`
       - Parse `result.discountAmountCents` from response and update state

    8. Update `handlePaymentSuccess()`:
       - After successful order creation and cart clearing, if `state.appliedCouponCode != null`:
         - Call `discountRepository.decrementCouponUsage(state.appliedCouponCode!!)`
         - Log failure but do NOT block navigation (known limitation per research: if decrement fails, coupon usage is not decremented)
       - Include discount fields in the Order object built for Room: `discountAmount = state.discountAmountCents / 100.0`, `discountCode = state.appliedCouponCode ?: ""`

    9. Update `OrderEntity` to include `discountAmount: Double = 0.0` and `discountCode: String = ""` fields. Update `OrderMapper` to map these fields between entity and domain model. NOTE: This is a non-destructive addition -- Room handles new fields with default values without a migration (fallbackToDestructiveMigration in debug). If a migration is needed, add MIGRATION_4_5 that ALTERs the orders table.
